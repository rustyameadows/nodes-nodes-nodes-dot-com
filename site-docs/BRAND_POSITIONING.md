# Nodes Nodes Nodes Brand Positioning

## Purpose
This document gives the splash page and the later marketing site one source of truth for product positioning, voice, approved labels, and status language.

## Product Summary
`Nodes Nodes Nodes` is a local-first desktop app for creative AI workflows. It gives one person a project-based workspace with an infinite canvas for building prompt and asset flows, running supported providers from one interface, and reviewing outputs in a comparison-oriented asset viewer.

Current product truths pulled from the reference docs:
- macOS desktop app
- local-first and single-user in v1
- project-based workflow with one canvas per project
- OpenAI image and text runs are supported
- Google Gemini image and text runs are supported
- Topaz image transforms are supported
- queue state and provider request-response debug data are visible
- asset review includes grid, 2-up, and 4-up comparison modes

## Audience
Primary audience:
- creative AI power users who actively run image and text workflows and want more control than provider-specific web UIs

Secondary audience:
- creative technologists and designers who need a serious local workspace for generation, review, and iteration
- developer-minded creators who care about workflow structure, provider visibility, and durable project state

## Positioning Statement
For creative AI power users who are tired of bouncing between prompts, provider tabs, and review tools, `Nodes Nodes Nodes` is a local-first desktop workspace that turns generation into a connected workflow. It combines a node canvas, durable runs, and fast visual review in one app so work stays organized from prompt to output.

## Messaging Pillars
### 1. Local-first by default
- Work happens in a desktop app, not a multi-tab browser stack.
- Projects, canvas state, jobs, and assets stay isolated locally.
- Messaging shorthand: `local-first desktop`

### 2. Workflow, not just prompting
- The app is about connecting notes, templates, assets, and model nodes on a canvas.
- The value is orchestration and iteration, not a single prompt box.
- Messaging shorthand: `creative AI workflows`

### 3. Review is part of the product
- Outputs are meant to be inspected, compared, tagged, rated, and filtered.
- The asset viewer is a core differentiator, not a secondary gallery.

### 4. One interface across supported providers
- The current product truth is OpenAI, Google Gemini, and Topaz in one workflow surface.
- Provider execution and job state are visible inside the app instead of being scattered across separate tools.

## Differentiators
- Infinite-canvas workflow builder instead of a linear chat or prompt form
- Local project isolation rather than one global history pile
- Integrated queue and job visibility
- Review-first asset workflows with compare modes
- Desktop app boundary that keeps filesystem and credentials out of the renderer surface

## Tone And Voice
Use this voice across splash-page and marketing copy:
- direct
- precise
- confident without startup hype
- visually minimal and serious
- creative-technical, not corporate

Prefer:
- short declarative sentences
- concrete product language
- verbs like `build`, `run`, `review`, `compare`, `iterate`
- phrases like `local-first desktop`, `creative AI workflows`, `project-based`, `comparison-oriented review`

Avoid:
- inflated claims like `revolutionary`, `game-changing`, `all-in-one for everyone`
- vague AI copy like `unlock creativity` or `supercharge innovation`
- consumer-app language that suggests a polished mass-market release

## Approved Labels
Standardize these exact outward-facing labels:
- `Nodes Nodes Nodes`
- `Join the waitlist`
- `Download the alpha`
- `macOS alpha`
- `local-first desktop`
- `creative AI workflows`

## Alpha Status Language
Use these facts consistently:
- `Nodes Nodes Nodes is a macOS alpha for creative AI workflows.`
- `It is currently built for a single local user.`
- `It combines a node canvas, provider runs, and output review in one desktop app.`
- `The waitlist is still a placeholder, and the current early-alpha handoff can point to the GitHub repo.`

Safe supporting language:
- `early access`
- `alpha`
- `in active development`
- `built for macOS first`
- `currently local-first and single-user`

## Do Not Claim
Do not claim any of the following unless the product docs change first:
- collaboration, shared workspaces, or multi-user editing
- cloud sync, hosted storage, or browser-based operation
- Windows or Linux support
- publicly available general release
- a live waitlist integration
- a polished self-serve public alpha install flow
- provider coverage beyond the currently documented support

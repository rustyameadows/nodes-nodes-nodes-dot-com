How it flows

A simple path through the app.

01

Open a project

Start fresh or pick up something older without mixing it into everything else.

02

Build on the canvas

Lay out notes, lists, templates, assets, and model blocks in a way that makes sense to you.

03

Run with your own providers

Send work through OpenAI, Google Gemini, or Topaz while keeping the job inside the same project.

04

Review and continue

Check the result, compare options, and carry the useful parts into the next pass.

What stays visible

Less guessing, less backtracking.

You know where the work lives

Projects keep their own workspace and outputs, which makes it easier to return to an idea later.

You can check a run when you need to

There is a clear place to see whether something is queued, running, done, or stuck.

You can compare without starting over

Review is part of the same flow, so strong results stay connected to the work that made them.

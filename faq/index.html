FAQ

The straightforward answers.

The basics

What it is and who it is for.

What is Nodes Nodes Nodes?

It is a local-first Mac app for building, running, and reviewing creative AI work in one place.

Who is it for?

People who want a steadier way to work than bouncing between provider tabs, folders, and scattered notes.

What makes it feel different?

It keeps the setup, the run, and the review close together, so the project is easier to follow over time.

What kinds of things can I work with?

You can build with notes, lists, templates, assets, and model blocks inside the same project.

Availability

Where it runs and how early it is.

Is it available right now?

Yes. The current version is a very early alpha that you get through GitHub.

Is it Mac only?

Right now, yes. The current alpha is focused on Apple Silicon Macs.

Does it run in the browser?

No. It is a desktop app.

Do I need an account?

No account is the point right now. The current product is focused on one person working locally on their own Mac.

Models and runs

What works today.

Which providers work today?

OpenAI, Google Gemini, and Topaz.

How do provider keys work?

You bring your own keys. In the app, you can add them in settings, and the GitHub repo covers the current early setup path too.

What is the Node Library?

It is a simple browse view for the main block types in the app, so you can understand the building blocks before you start laying out a canvas.

Can I see what happened when a run goes wrong?

Yes. The app keeps runs visible so you can check their status and go back to what happened.

Limits right now

What is still early or missing.

Does it support collaboration or cloud sync?

Not yet. The current version is local-first and single-user.

Is the release flow polished yet?

No. The app itself is the focus right now, which is why the current alpha is distributed through GitHub.

Does local-first mean my work stays on my Mac?

Yes. Your projects, workspace state, and outputs are built around a local desktop setup.

Where should I look for privacy details?

The privacy page has the current plain-language overview while the fuller policy is still being finalized.
